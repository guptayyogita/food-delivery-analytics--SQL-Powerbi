# food-delivery-analytics--SQL-Powerbi
Food Delivery Performance &amp; Customer Behavior Analytics  SQL + Power BI project analyzing delivery efficiency, customer spending patterns, and operational performance in a food delivery platform similar to Swiggy/Zomato.

Project Overview
This project analyzes food delivery operations to identify key factors affecting delivery performance and customer behavior.

The analysis focuses on answering important business questions such as delivery delays, peak ordering hours, high-value customers, and how delivery time impacts customer ratings.

The project demonstrates end-to-end data analytics workflow including SQL analysis, data modeling, and interactive dashboard development.

Business Problem

Food delivery companies need to monitor operational performance and customer satisfaction.

This project answers key business questions:
• Which cities experience the highest delivery delays?
• What are the peak order hours?
• Which customers generate the most revenue?
• How does delivery time impact ratings?
• Which payment modes generate the most revenue?

Tools & Technologies Used
• SQL (Joins, CTEs, Window Functions)
• Microsoft Power BI (Data Modeling, DAX, Dashboarding)

Key Analysis Performed
SQL Analysis

• Multi-table joins to build complete order dataset
• Late delivery percentage by city
• Peak ordering hours analysis
• Top customers by total spending using window functions
• Running revenue trend analysis

Power BI Dashboard
Built an interactive dashboard including:
• Executive KPI overview
• Delivery performance analysis
• Customer behavior insights
• Peak hour order trends

Key Insights
• 8 PM – 12 PM is the peak order period
• Longer delivery distance increases delay probability
• Faster deliveries lead to higher customer ratings
• A small group of customers contributes significantly to revenue
